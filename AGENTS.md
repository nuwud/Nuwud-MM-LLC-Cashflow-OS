# AGENTS.md — Cashflow OS Rules

This repository exists to create cashflow for Nuwud Multimedia LLC.

## Core identity

Nuwud Multimedia LLC is an AI-native systems studio for founder-led brands. It does not primarily sell websites. It designs revenue infrastructure: ecommerce systems, automation, AI workflows, analytics, conversion architecture, founder dashboards, and reusable operating systems.

## Prime directive

Prefer boring, repeatable revenue motion over clever, unfinished assets.

Every contribution should answer at least one of these:

1. Does this help identify better prospects?
2. Does this start or improve a sales conversation?
3. Does this help convert a conversation into a paid diagnostic?
4. Does this help collect money sooner and more professionally?
5. Does this standardize delivery so Patrick is not reinventing work each time?
6. Does this create a reusable asset, proof point, or automation that compounds?

## Decision filter

Classify work as one of:

- Revenue Now: moves money in 0–14 days.
- Revenue Soon: builds pipeline in 15–45 days.
- Asset: supports sales but does not directly sell.
- Distraction: feels productive but avoids the money conversation.

Prioritize Revenue Now, then Revenue Soon, then Assets. Cut Distractions.

## Rules for automations

- Do not automate chaos.
- Do not create spam systems.
- Respect platform rules, opt-outs, disclosures, and human review.
- All outbound content should be value-led and reputation-safe.
- Any affiliate mention must be clearly disclosed.
- Affiliate links should never replace the primary service offer unless the post is intentionally educational or tool-focused.

## Rules for offer development

- Keep the paid diagnostic as the front door.
- Keep scope small enough to sell and deliver quickly.
- Never give away the full implementation plan before payment.
- Use audits to create implementation proposals.
- Document repeatable findings as reusable IP.

## Definition of done

A file or workflow is done only when it can be used in the next sales or delivery action without additional strategy work.
