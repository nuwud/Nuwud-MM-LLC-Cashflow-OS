# Affiliate Activation OS

This file turns affiliate research into an executable cashflow lane for Nuwud Multimedia LLC.

Affiliate revenue is useful only when it reinforces the main business:

```text
Authority -> Conversation -> Paid Audit -> Implementation -> Retainer -> Proof -> Affiliate/Partner Upside
```

The goal is not to become an affiliate marketer.

The goal is to make Nuwud's tool-stack recommendations, audits, tutorials, and resource pages financially useful while strengthening trust.

---

## Non-negotiable rule

This repository is public.

Do **not** store any of the following in GitHub:

- passwords
- recovery codes
- tax forms
- SSN / EIN paperwork
- banking details
- PayPal account screenshots
- Stripe dashboard screenshots
- affiliate dashboard screenshots with private IDs
- private affiliate links before they are ready to be published
- client private data

Use GitHub for operating docs, prompts, public signup URLs, program status, and content templates.

Use a password manager, private vault, or private Notion database for sensitive account details.

## Browser AI signup workflow

Use browser AI as an application assistant, not as an unsupervised account owner.

### Browser AI may help with

- opening affiliate program pages
- summarizing program terms
- filling public profile fields
- drafting application answers
- checking that Nuwud positioning is consistent
- creating a tracker row
- listing required follow-up actions

### Browser AI must not do without Patrick approval

- submit final applications
- accept legal terms
- enter banking information
- enter tax details
- create passwords
- change account security settings
- publish affiliate links
- send emails as Patrick

### Application identity pack

Use this language as the consistent application foundation.

```text
Nuwud Multimedia LLC is an AI-native systems studio for founder-led brands. We design revenue infrastructure across Shopify, WordPress, automation, AI workflows, analytics, conversion systems, and founder dashboards.

Our audience includes founders, ecommerce operators, service businesses, and small teams that need clearer digital systems, better follow-up, stronger websites, and more reliable revenue operations.

We plan to promote partner tools through educational content, audit/resource pages, implementation guides, tool-stack recommendations, tutorials, and client-facing system recommendations where the tool is a genuine fit.

We do not run coupon sites, spam campaigns, misleading income claims, or undisclosed affiliate promotions.
```

### Promotion plan answer

Use this when a program asks how Nuwud will promote it.

```text
We will promote this program through educational, value-led content connected to real business problems: revenue leaks, website performance, ecommerce conversion, CRM/follow-up gaps, automation opportunities, and founder operating systems.

Promotions may include LinkedIn posts, X posts, blog/resource pages, tutorial videos, implementation checklists, and audit resource recommendations. Affiliate relationships will be disclosed clearly. We will not use deceptive urgency, coupon-style positioning, spam, or unapproved paid ad tactics.
```

## Affiliate post rules

Every affiliate post must pass this checklist before publishing.

- The post teaches something useful even if nobody clicks.
- The tool is connected to a business problem.
- The recommendation sounds like Patrick's judgment, not vendor copy.
- The disclosure is visible and plain.
- The CTA points to a system, audit, checklist, or resource — not only a signup link.
- No income claims.
- No fake urgency.
- No private client details.
- No unsupported claims about performance or guaranteed results.
- No affiliate link unless the program is approved and the tracking link is correct.

## Revenue expectation

Affiliate revenue should be treated as a compounding secondary layer.

The immediate money path remains paid audits, sprints, buildouts, and retainers.

Affiliate income becomes meaningful when attached to:

- recurring authority content
- client implementation recommendations
- resource pages
- tutorials
- comparison pages
- repeatable audit findings
- templates and workflow libraries

The highest-leverage version is not random posting.

It is this:

```text
Audit finding -> educational post -> recommended tool -> implementation offer -> retainer/support path
```
